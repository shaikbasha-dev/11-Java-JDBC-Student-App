# Code of Conduct

## Our Commitment

The **11-Java-JDBC-Student-App** repository is committed to providing a welcoming, respectful, and inclusive learning environment for everyone, regardless of experience level, background, identity, or skill level.

This repository is intended to support students, beginners, educators, and software developers who want to learn Java, JDBC, MySQL, and professional software development practices.

Every community member deserves to be treated with respect and professionalism.

---

# Purpose

The purpose of this Code of Conduct is to:

- Promote a positive learning environment.
- Encourage respectful collaboration.
- Support constructive discussions.
- Foster knowledge sharing.
- Maintain a professional open-source community.
- Ensure all contributors feel welcome.

---

# Expected Behavior

Community members are encouraged to:

- Be respectful and professional.
- Communicate politely and constructively.
- Help beginners learn and grow.
- Share knowledge openly.
- Accept constructive feedback gracefully.
- Give helpful suggestions.
- Respect different opinions and perspectives.
- Focus discussions on improving the repository.
- Report issues responsibly.

---

# Unacceptable Behavior

The following behaviors are not acceptable:

- Harassment or discrimination.
- Personal attacks or insults.
- Offensive or abusive language.
- Trolling or intentionally disruptive behavior.
- Publishing private or confidential information.
- Spam or promotional content unrelated to the repository.
- Plagiarism or copyright violations.
- Misleading technical information intended to confuse learners.

Such behavior negatively impacts the learning experience and will not be tolerated.

---

# Respectful Communication

When participating in discussions, issues, or pull requests:

- Be patient.
- Be courteous.
- Assume good intentions.
- Provide constructive feedback.
- Explain technical suggestions clearly.
- Respect differing viewpoints.
- Focus on solving problems rather than criticizing individuals.

---

# Constructive Feedback

Helpful feedback should:

- Be specific.
- Be respectful.
- Be solution-oriented.
- Include technical reasoning where appropriate.
- Encourage learning and improvement.

Examples include:

- Suggesting code improvements.
- Recommending clearer documentation.
- Identifying potential bugs.
- Explaining better implementation approaches.

---

# Reporting Issues

If you encounter inappropriate behavior:

1. Document the issue.
2. Provide relevant details.
3. Report the behavior through GitHub Issues or another appropriate communication channel.
4. Allow the repository maintainer to review the situation.

All reports will be reviewed fairly and respectfully.

---

# Maintainer Responsibilities

Repository maintainers are responsible for:

- Maintaining a welcoming environment.
- Reviewing issues and pull requests fairly.
- Enforcing this Code of Conduct consistently.
- Encouraging respectful collaboration.
- Removing inappropriate or harmful content when necessary.
- Protecting the educational purpose of the repository.

---
