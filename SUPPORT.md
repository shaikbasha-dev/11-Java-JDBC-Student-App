# Support

## Introduction

Thank you for visiting the **11-Java-JDBC-Student-App** repository.

This project has been created as an educational resource to help students, beginners, and aspiring Java developers understand Java Database Connectivity (JDBC), MySQL integration, and object persistence through practical examples.

If you need assistance while using this repository, the information below explains the best ways to get help.

---

# Before Requesting Support

Before creating an issue or requesting assistance, please verify the following:

- You have carefully read the main `README.md`.
- You have reviewed the documentation inside the `src` directory.
- You have followed the database setup instructions.
- You have verified your MySQL configuration.
- You have checked the troubleshooting guide.
- You have searched existing GitHub Issues to see whether your question has already been answered.

Many common questions can be resolved by reviewing the existing documentation.

# Reporting Issues

If you encounter a problem that is not already documented, please create a GitHub Issue.

When reporting an issue, include:

- A clear and descriptive title.
- A detailed explanation of the problem.
- Steps to reproduce the issue.
- Expected behavior.
- Actual behavior.
- Error messages or stack traces.
- Screenshots (if applicable).
- Java version.
- MySQL version.
- Operating system.

Providing complete information helps reproduce and resolve issues more efficiently.

# Self-Help Checklist

Before asking for support, verify the following:

- MySQL Server is running.
- Database `test` exists.
- Table `sample` exists.
- MySQL Connector/J has been added.
- JDBC URL is correct.
- Username and password are correct.
- Java source code compiles successfully.
- No SQL syntax errors exist.
- Database records have been verified using `SELECT * FROM sample;`

# Best Way to Learn

To gain the maximum benefit from this repository, follow this learning sequence:

1. Read the main README.
2. Understand the project objective.
3. Study the source code.
4. Review each documentation file in the `src` directory.
5. Configure the database.
6. Execute the application.
7. Verify the database output.
8. Review the FAQ.
9. Practice the interview questions.
10. Experiment by modifying the project.

Hands-on practice is the most effective way to understand JDBC.
